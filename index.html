제가 맡아온 일

신규 화면 개발, 기존 기능 고도화, 데이터 오류 처리, 장애 원인 분석, 배포 후 운영 확인을 수행했습니다.
요청받은 기능을 만드는 것에서 끝내지 않고, 실제 사용자가 어떤 순서로 조회·입력·저장하는지와 데이터가 맞게 남는지까지 확인합니다.

STRENGTH 01운영 이슈를 구조화합니다DB Timeout, WAS 다운, 대용량 조회 지연처럼 원인이 바로 보이지 않는 문제를 로그, SQL, 데이터 흐름 기준으로 쪼개어 확인합니다.
STRENGTH 02사용자 업무 흐름을 반영합니다병원 실무자가 어떤 순서로 조회, 입력, 저장하는지 확인하고 화면 흐름과 입력 기준을 정리해 업무 혼선을 줄입니다.
STRENGTH 03DB와 데이터 정합성을 중시합니다SQL 성능, 마이그레이션, Job, 연동 기준을 함께 보며 화면 결과와 실제 데이터가 맞는지 확인합니다.
STRENGTH 04협업 기준을 문서화합니다타 팀 연동 항목, 처리 시점, 상태값, 운영 처리 기준을 정리해 이후 유지보수자가 같은 기준으로 볼 수 있게 합니다.

문제를 해결하는 방식

01현상 확인 · 문의·로그·데이터 확인
02원인 분리 · 화면·SQL·서버 구간 분리
03개선 반영 · 쿼리·로직·설정 수정
04재발 방지 · 검증·문서화

제가 지향하는 개발

기능이 동작하는 것보다 중요한 것은 운영 중에도 흔들리지 않는 구조라고 생각합니다.
ERP 환경에서는 작은 오류도 사용자 업무와 데이터에 영향을 줄 수 있기 때문에, 문제를 임시로 덮기보다 원인을 확인하고 기준을 남기는 방향으로 일합니다.
